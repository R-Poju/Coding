前置声明

避免 #include 死锁，解决循环引用。当两个类互相引用时，直接 #include 会导致编译错误。前置声明可打破循环依赖：

```c++
// CSession.h
class CServer;  // 前置声明（无需包含 CServer.h）

class CSession {
    CServer* m_server;  // 合法：指针/引用只需知道类名存在
    void connectToServer(CServer* srv); 
};
```

减少头文件解析范围，提升编译速度

头文件不暴露无关依赖，降低耦合度





什么情况下必须用 #include ？

1.调用CServer的成员函数

```c++
void CSession::attachToServer(CServer* srv) {
    srv->start();  // 需要 CServer 的完整定义！
}
```

2.计算类大小或内存对齐

```c++
sizeof(CServer);	//需完整定义
```
